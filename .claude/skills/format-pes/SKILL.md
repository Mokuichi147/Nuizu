# Brother PES / PEC フォーマット仕様

PESはBrother/Babylock刺繍機向けのハイブリッド形式。
PESセクション（デザインデータ）+ PECセクション（マシンコマンド）で構成。

出典: [EduTechWiki PES](https://edutechwiki.unige.ch/en/Embroidery_format_PES), [EduTechWiki PEC](https://edutechwiki.unige.ch/en/Embroidery_format_PEC), [pyembroidery](https://github.com/EmbroidePy/pyembroidery)

## PES ファイル構造

```
[PESヘッダ] [PESデザインデータ(オプション)] [PECセクション]
```

### PESヘッダ

| オフセット | サイズ | 内容 |
|-----------|-------|------|
| 0 | 4 | マジック `#PES` (ASCII) |
| 4 | 4 | バージョン文字列: `0001`, `0020`, `0030`, `0040`, `0050`, `0060` 等 |
| 8 | 4 | PECセクションへのオフセット (u32 LE) |

全数値はリトルエンディアン。座標単位は **0.1mm**。

### バージョン別の違い

| バージョン | 特徴 |
|-----------|------|
| v1 (0001) | 最小構成。12バイトで有効なファイルが作れる |
| v2 (0020) | UI設定パラメータ追加 |
| v4 (0040) | 説明文、幾何オブジェクト対応 |
| v5/v6 (0050/0060) | カスタム糸色（64色PECパレット非依存）、フィルパターン、アフィン変換 |

### 最小有効PESファイル (v1)

```
#PES0001              (8バイト: マジック+バージョン)
[PECオフセット: u32]   (4バイト: = 0x16 等)
[ゼロパディング]       (10バイト)
[PECセクション]
```

## PEC セクション構造

```
[ラベル 20B] [パディング 12B] [固定値 2B] [サムネイルパラメータ 2B]
[パディング 12B] [色数 1B] [色インデックステーブル] [パディング]
[ステッチブロックヘッダ 16B] [ステッチデータ] [サムネイル画像]
```

### PECラベル (20バイト)

```
"LA:" + 名前(最大16文字、0x20パディング) + 0x0D
```

### 色テーブル

| オフセット(PEC内) | サイズ | 内容 |
|-----------------|-------|------|
| 20 | 12 | `0x20` パディング |
| 32 | 1 | `0xFF` |
| 33 | 1 | `0x00` |
| 34 | 1 | サムネイル幅バイト数（通常 `0x06` = 48px） |
| 35 | 1 | サムネイル高さ（通常 `0x26` = 38px） |
| 36 | 12 | パディング `0x20` |
| 48 | 1 | 色数 - 1 |
| 49 | N | 色インデックス（PEC 64色パレットへの参照） |
| 49+N | 463-N | `0x20` パディング（合計463エントリ） |

### ステッチブロックヘッダ (16バイト)

| オフセット | サイズ | 内容 |
|-----------|-------|------|
| 0 | 2 | `0x00 0x00` |
| 2 | 3 | ブロック長（uint24 LE） |
| 5 | 3 | `0x31 0xFF 0xF0` (マジック) |
| 8 | 2 | デザイン幅 (s16 LE, 0.1mm) |
| 10 | 2 | デザイン高さ (s16 LE, 0.1mm) |
| 12 | 2 | `0x01E0` (480) |
| 14 | 2 | `0x01B0` (432) |

## PEC ステッチエンコーディング

座標単位: **0.1mm**

### ショートフォーム (1バイト/座標)

```
ビット: 0bXXXXXXX  (bit7 = 0)
値: 7ビット2の補数 (-64 ~ +63)
```

1ステッチ = 2バイト (dx + dy)。

### ロングフォーム (2バイト/座標)

```
ビット: 0b1CCC_BBBB BBBBBBBB  (bit7 = 1)
CCC: コントロールコード
B(12bit): 2の補数 (-2048 ~ +2047)
```

1ステッチ = 4バイト (2バイト×dx + 2バイト×dy)。

### コントロールコード (CCC)

| CCC | コマンド | ビットパターン |
|-----|---------|--------------|
| 000 | 通常ステッチ（ロング） | `0b1000xxxx` |
| 001 | ジャンプ | `0b1001xxxx` |
| 010 | トリム | `0b1010xxxx` |

フラグ定数:
- `JUMP_CODE = 0x10` (bit4)
- `TRIM_CODE = 0x20` (bit5)

### 特殊コマンドバイト値

| コマンド | バイト列 |
|---------|---------|
| 色替え | `0xFE 0xB0` + 交互に `0x02` / `0x01` |
| END | `0xFF` |

色替えの第3バイトは `(色替え番号 % 2) + 1` で交互に変わる。

### 最大移動距離

- ショートフォーム: ±63 (±6.3mm)
- ロングフォーム: ±2047 (±204.7mm)
- 超過時は分割して複数コマンドに

## PEC カラーパレット (64色)

| Index | RGB | 色名 |
|-------|-----|------|
| 1 | (14,31,124) | Prussian Blue |
| 2 | (10,85,163) | Blue |
| 5 | (237,23,31) | Red |
| 13 | (255,255,0) | Yellow |
| 20 | (0,0,0) | Black |
| 29 | (240,240,240) | White |

（全64色。Index 0 は未使用/不明）

## サムネイル

- サイズ: 48 x 38 ピクセル (= 6バイト x 38行 = 228バイト)
- 1ピクセル1ビット、MSBファースト
- 全体サムネイル1枚 + 色ごとに1枚
- 空サムネイル（全`0x00`）でも動作する
